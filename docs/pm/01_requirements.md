## 1. プロダクト概要

- **正式名称**: TANREN(鍛錬)— 2026-07-10 正式決定(DEC-001)
- **一言で言うと**: 手持ち器具と過去ログから「今日の最適メニュー」を生成する、Eiichi専用の家トレコーチPWA
- **利用者**: Eiichi 1名(認証なし・非公開URL)
- **利用シーン**: 自宅(ベンチ台+可変ダンベル)。スマホ片手にトレしながら使う

### 解決する課題

1. 不定期トレでも「今日は何をどれだけやるべきか」を考えなくていい
2. 前回の重量・レップを覚えていなくても漸進的に強くなれる
3. インターバルの管理(何秒休むか+計測)を丸投げできる
4. 目標ボディとの差分・成長が記録から見える

## 2. スコープと前提

| 項目 | 決定内容 |
|------|----------|
| プラットフォーム | PWA(スマホブラウザ→ホーム画面追加) |

## 5. 機能要件

### F-01 初期セットアップウィザード

1. プロフィール入力(身長・体重・任意で体脂肪率)
2. **目標ボディ選択**(アプリ内でいつでも変更可):
   - 細マッチョ(引き締め+適度な筋量)
   - バルクアップ(筋量最大化)
   - 体力・健康維持
   - 部位特化(特定部位を選んで集中強化)
3. **現状写真の登録**(任意・スキップ可): 正面/横/背面
4. **ヒヤリング**:
   - 鍛えたい部位(複数選択: 胸・背中・肩・腕・脚・腹・尻)
   - 鍛えたくない部位(複数選択+理由タグ: 怪我/好みでない/十分発達)
   - 怪我・違和感のある箇所(自由記述)

### F-02 器具設定

- デフォルトで第4章のEiichi器具をシード
- 器具の追加・編集・無効化が可能(将来: ジム用/出張用プロファイル)
- ダンベルの重量ステップは配列で管理(重量提案エンジンがこの刻みにスナップする)

### F-03 ギャップ分析とベース方針

- 目標ボディテンプレートごとに部位優先度マップを定義(例: 細マッチョ=肩・胸上部・腹・背中広がり重視)
- ヒヤリング結果(鍛えたい/避けたい)を掛け合わせて**部位別優先度スコア**を算出
- 出力: 「あなたの優先部位トップ3と理由」「週あたり推奨部位別セット数(ボリューム目標)」
- 不定期前提のため固定分割表は作らず、F-04が毎回このスコアを参照する

### F-04 その日のメニュー生成(コアエンジン)🔴最重要

**入力(その日のヒヤリング・3タップ以内で完了)**:
1. 今日使える時間: 15 / 30 / 45 / 60 / 90分
2. 鍛えたい部位: 指定 or おまかせ
3. コンディション: 絶好調 / 普通 / 疲れ気味

**生成ロジック**:
1. **回復モデル**: 部位ごとに直近セッションの経過時間×ボリュームから筋フレッシュネス(0-100%)を算出。基準回復時間は大筋群72h/小筋群48h
2. **部位決定**: おまかせ時は「優先度スコア × フレッシュネス」最大の部位群を選択。指定時もフレッシュネス不足なら警告
3. **種目選択**: 器具フィルタ→コンパウンド優先→時間枠にセット数が収まるよう構成(1セット+インターバル+移行 ≈ 実測ベースで見積り)
4. **重量・レップ提案**: 前回ログからダブルプログレッション(レップ上限到達→ダンベル次ステップへ増量してレップ下限から再開)。初回種目は自己申告+控えめ開始
5. **インターバル指定**: 種目ごとに目的別テーブルから付与
   - 筋力(1-5レップ): 180秒
   - 筋肥大(6-12レップ): 90秒
   - 持久・引き締め(13+レップ): 60秒
   - コンパウンド種目は+30秒
6. **コンディション補正**: 疲れ気味→総ボリューム-20%・高重量種目を回避 / 絶好調→PR挑戦セットを1つ提案
7. 生成後、種目の**入れ替え・削除・追加が可能**(代替種目は同部位・同器具から提示)

### F-05 ワークアウト実行画面

- 種目カード縦スクロール。現在の種目がハイライト
- セット行: 提案重量・提案レップ表示 → 実績を入力(±ボタンで素早く)→ 完了チェック
- **インターバルタイマー**:
  - セット完了チェックで自動起動(ON/OFF設定可)
  - 大きな円形カウントダウン表示、残り3秒でカウント音、終了でバイブ+音
  - ±15秒調整、スキップ可
  - タイマー中も画面消灯しない(Wake Lock)
- 種目単位のメモ欄+セッション全体のメモ欄
- 中断保存(途中でやめても記録が残る)

### F-06 ログ記録項目(メモの標準フォーマット)🔴Eiichi指定要件

セッションごとに以下を記録する。※=自動記録、それ以外は入力(全て任意・後から追記可):

| 項目 | 内容 |
|------|------|
| 実施日時・所要時間 ※ | 開始/終了から自動算出 |
| 種目・セット・重量・レップ ※ | 実行画面の入力から |
| 完遂率 ※ | 提案メニューに対する実施率 |
| RPE(キツさ1-10) | セッション全体+任意で種目別 |
| できた/できなかった | 提案重量・レップに対する達成フラグ(次回提案に直結) |
| 体調メモ | 睡眠・食事・気分など自由記述 |
| 痛み・違和感フラグ | 部位選択+メモ(次回メニューで該当部位を自動回避) |
| 次回への申し送り | 自由記述(次回メニュー生成時に画面表示) |

### F-07 分析・振り返り

- **セッション後サマリー**: 前回比(重量/レップ/ボリューム)、PR検出(「ダンベルプレス 20kg×10 自己新!💪」)、部位別今週ボリューム
- **ダッシュボード**: 部位別週間ボリューム推移グラフ、筋フレッシュネスマップ(人体図)、体重推移
- **写真比較ビュー**: 過去写真と最新を並べて表示、日付スライダー
- **毎回の現状分析**(ルールベース): ログから「胸は順調に伸びてます。脚が2週間空いてるので次回候補」等のコメント生成

### F-08 データ管理

- JSON一括エクスポート/インポート(写真含む・機種変とバックアップ用)
- 全データ削除(誤操作防止の二段確認)

### F-09 AIモード(Phase 3・オプション)

- 設定画面にAnthropic APIキー入力欄(ローカル保存)
- 有効時: 写真の変化コメント、自然言語でのヒヤリング、メニューへのAIひとことアドバイス
- 無効時も全機能が完結すること(AIは添え物という設計思想)

---

## 6. 非機能要件

- オフライン完結(機内モードでもトレ可能)
- モバイルファースト(iPhone Safari基準)。片手・ワンタップ操作優先
- メニュー生成は体感即時(<300ms)
- エンジンロジック(回復モデル/重量提案/インターバル)はユニットテスト必須

## 7. データモデル(概要)

profiles / goals / equipment / exercises(種目マスタ) / sessions / session_exercises / sets / photos / body_stats / injuries

- 種目マスタ初期データ: **ダンベル+アジャスタブルベンチ+自重で35種目前後**を部位タグ・必要器具・compound/isolation・推奨レップ帯付きで用意(Engineerが作成、PMがレビュー)
