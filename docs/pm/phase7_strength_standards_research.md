# Phase 7 調査: 筋力標準と3レベル係数表草案(DEC-013候補・§3-1)

- 作成: PM / 2026-07-23
- 前提: `docs/pm/phase7_goal_model_concept.md`(方式A・体重連動・鏡チェックループは決定済み)
- 目的: 部位別ゴール「引き締め / しっかり / がっつり」の裏数値(目標e1RM = 体重 × 係数)の根拠を作る

---

## 1. 採用ソース

**主ソース: Strength Level(strengthlevel.com)**
- クラウドソース標準。種目ごとに数十万〜数百万リフトのログから百分位を算出(例: DBベンチプレスは470万リフト超)
- レベル定義が百分位で明確: Beginner=上位95%外(5%より強い)/ Novice=20% / Intermediate=50% / Advanced=80% / Elite=95%
- **ダンベル種目の標準が片手1RM表記**で揃っており、TANRENのe1RM(片手・Epley)とそのまま突合できる
- 体重比・体重別・年齢別のテーブルが全種目にある(kg表記あり)

補助ソース(妥当性確認用): FitnessVolt(OpenPowerliftingの実測250万件基盤)、Endura。傾向は概ね一致。実装はStrength Level一本に寄せて出典を単純化する。

**注記(片手1RMとの整合)**: Strength Levelのダンベル値は「片手のダンベル重量」。TANRENのe1RMも片手なので変換不要。ソースによっては両手合計表記(Enduraなど)があるため、将来数値を差し替える場合は表記単位に注意。

## 3. TANREN 3レベルへのマッピング提案

| TANRENゴール | 対応レベル | 根拠 |
|--------------|-----------|------|
| 引き締め | **Novice**(上位80%) | 「うっすら形がある」。半年程度の定期トレで届く現実的な入口 |
| しっかり | **Intermediate**(上位50%) | 「鍛えているとわかる」。トレーニング人口の中央値=一般人比では明確に発達 |
| がっつり | **Advanced**(上位20%) | 「明らかに大きい」。数年スパンの到達目標として機能する |

- **Eliteは採用しない**: 上位5%は家トレ・ダンベル24kg上限の環境と噛み合わず、ゴールとして非現実的。「がっつり到達後にまだ物足りない」場合は数値直接編集で対応(方式Aの裏数値編集がそのまま逃げ道になる)
- 係数はStrength Levelの体重比をそのまま使う(独自の丸めをしない。出典との対応が追える方が調整時に楽)

### 係数表草案 v0(男性・目標e1RM = 体重 × 係数・片手)

| 部位 | 引き締め | しっかり | がっつり |
|------|---------|---------|---------|
| 胸 | 0.35 | 0.50 | 0.70 |
| 背中 | 0.35 | 0.55 | 0.75 |
| 肩 | 0.25 | 0.40 | 0.55 |
| 腕 | 0.15 | 0.30 | 0.50 |
| 脚 | 0.25 | 0.40 | 0.60 |

体重70kgの例: 胸のしっかり=片手e1RM 35kg、腕のがっつり=35kg、肩の引き締め=17.5kg。ダンベル上限24kg×高レップのe1RM換算(Epley・レップ12クランプで最大+40%≒33.6kg)を考えると、**胸・背中の「がっつり」(49〜52.5kg)は現器具では到達不能**。これは仕様バグではなく「器具の上限が先に来る」という事実で、UIで正直に扱う(§6論点C)。

## 4. 年齢の扱い

Strength Levelの年齢別テーブルは**全種目でほぼ同一の比率カーブ**だった(ピーク25〜40歳)。種目ごとに持たず、共通の年齢係数1本で持てる:

| 年齢 | 係数(ピーク比・5種目平均) |
|------|--------------------------|
| 15 | 0.87 |
| 20 | 0.97 |
| 25–40 | 1.00 |
| 45 | 0.95 |
| 50 | 0.89 |
| 55 | 0.82 |
| 60 | 0.75 |
| 65 | 0.68 |
| 70 | 0.61 |
| 75+ | 0.55以下(5歳ごとに約0.9倍) |

- 目標e1RM = 体重 × 部位係数 × 年齢係数
- v1実装は**年齢係数を定数1.0で開始してもよい**(Eiichiは現在ピーク帯のはず。設定に生年月日を持つかは論点D)

## 5. 身長の扱い(構想§3の検証課題)

**結論: 数値には組み込まない**。調査したすべての標準(Strength Level / FitnessVolt / Endura)は体重比ベースで、身長は変数に含まれない。身長はリーチ差として個人差に吸収されており、公開データで補正する根拠がない。構想どおり説明文側(「見え方は体脂肪率や骨格でも変わります」)で扱う。

## 6. 残る論点(Eiichi判断)

- **A. 尻の扱い**: ダンベル種目の尻の公開標準が薄い(ヒップスラスト等はバーベル基準)。案: **v1は尻をゴールモデル対象外**とし、現行の固定優先度のまま。将来ログが貯まったら実績ベースで自前基準を作る
- **B. 腹の扱い**: レップベース標準は存在確認済み(Strength Levelのクランチ: 平均男性51レップ=Intermediate)。ただしe1RMとは別系統の実装になる。案: **v1は腹も対象外**(尻と同じ理由+実装を軽く保つ)。対象は胸・背中・肩・腕・脚の5部位で開始
- **C. 器具上限とがっつり**: 胸・背中のがっつりは24kgダンベルでは物理的に届かない。案: ゴール設定UIで到達不能ゴールに「現在の器具では届きません」注記を出す(選択自体は妨げない)
- **D. 年齢係数**: v1から入れるか、定数1.0で開始するか
