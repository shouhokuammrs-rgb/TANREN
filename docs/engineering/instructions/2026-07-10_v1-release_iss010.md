# Instruction: v1.0 Release — ISS-010 本番ホスト判定の修正

- Priority: 🔴(本番URLに常時警告が出る偽陽性)
- 発行: PM / 2026-07-10
- 対応Issue: ISS-010

## 事象

実際の本番ドメインは `tanren.vercel.app` ではなく Vercel が割り当てた別名(Eiichi確認値: 指示時に伝達。例 `tanren-lake.vercel.app`)。ISS-009のプレビューURL警告がハードコードされた `tanren.vercel.app` と比較しているため、本番で警告バナーが常時表示される。

## 要件

1. 本番ホスト名をハードコードせず、ビルド時環境変数 `VITE_PROD_HOST`(vercel.json/Vercelの環境変数で注入)から取得。未設定時は警告機能を無効化(偽陽性より無警告の方がマシ)
2. Eiichiから伝達された正式ドメインをリポジトリのデフォルト値として設定
3. 判定ユニットテストを新ドメインで更新(偽陽性ケース: 本番ドメイン自身、偽陰性ケース: `*-git-*.vercel.app` などのプレビューURL)
4. ついでにREADMEに本番URL・リリース手順(タグ運用)を1セクション追記
