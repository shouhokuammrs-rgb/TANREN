# Instruction: 定常運転 #3 — バックアップ体制の強化(最優先)

- Priority: 🔴
- 発行: PM / 2026-07-17
- 前提: main最新(PR #8マージ後)
- 対応Issue: ISS-014

## 背景(インシデント2件目)

ホーム画面のPWAアイコン削除により、iOS仕様でオリジンのIndexedDBごと消去され、全トレ記録を喪失(復旧不能)。
初日のプレビューURL事件に続く2件目のデータ喪失。原因はiOS仕様であり技術的に防げないため、
**「エクスポートJSONを外部に置く習慣を、アプリ側から半強制的に作る」**方向で対策する。

## 要件

### 1. バックアップ導線の常設・多重化
- トレ終了サマリー画面の末尾に「💾 バックアップする」ボタンを常設(データが増えた直後が最重要タイミング)
- 初回セットアップウィザード完了画面にも同ボタン+「iCloud Driveなど端末の外への保存を推奨」の1行
- 既存の7日リマインダーは維持しつつ、**未バックアップのセッションが3件溜まったら**ホームのリマインダーを強調表示(件数表示付き)に格上げ

### 2. 危険操作の警告明示
- 設定のデータ管理セクションに注意書きを常設: 「⚠️ ホーム画面のアイコンを削除すると、記録もすべて削除されます(iOSの仕様)。エクスポートしたJSONだけが復元手段です」
- 全データ削除の確認ダイアログにも同趣旨を追記

### 3. エクスポート体験の摩擦低減
- 現在の共有シート起動に加え、ファイル名を `tanren-backup-YYYYMMDD.json` 形式に統一(復元時にどれが最新か一目で分かる)
- エクスポート完了時にトースト表示+「最終エクスポート」日時を即時更新
- インポート画面に「最後のエクスポート以降の記録は戻りません」の説明を追加

## 受け入れ条件

- [ ] サマリー/ウィザード完了/設定の3箇所からエクスポートが起動できる(実機)
- [ ] 未バックアップ3セッションで強調リマインダーが出る(E2E・日時注入)
- [ ] 警告文言が設定と削除ダイアログに表示される
- [ ] テストgreen、mainへのPR作成まで

## スコープ外(将来ISS候補として記録のみ)
