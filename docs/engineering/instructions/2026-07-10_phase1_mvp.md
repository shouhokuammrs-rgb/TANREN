# Instruction: Phase 1 — MVP「今日から使える」

- Phase: 1
- Priority: 🔴
- 発行: PM / 2026-07-10
- 前提: Phase 0マージ済み(main)。要件 `docs/pm/01_requirements.md` §5 F-02/F-04/F-05/F-06 が対象

## 目的

「3タップでメニュー生成 → トレ実行(タイマー付き) → ログ保存」の一連が実機で使える状態にする。Eiichiが実トレで使うことが受け入れテスト。

## スコープ(WBS 1-1〜1-6)

### 1-1 種目マスタ35種目前後のシード
- ダンベル+アジャスタブルベンチ(-20°〜90°)+自重で構成
- 各種目: 部位タグ(胸/背中/肩/腕/脚/腹/尻)、必要器具、compound/isolation、推奨レップ帯、ベンチ角度(該当時)
- 完了後、一覧をhandoffに添付(PMが種目構成をレビューする)

### 1-2 器具設定の編集+ヒヤリングウィザード🆕(DEC-002差し替え)
- ダンベル編集モーダル: 「最小重量」「最大重量」「段階数」の3問 → 等間隔配列を自動生成(0.5kg丸め)→ 生成一覧を表示し、各値をタップで個別修正可能
- 設定画面の「DEC-002仮設定」注記は削除
- ベンチ: 角度範囲の編集可

### 1-3 メニュー生成エンジン v1(要件F-04のロジック1〜7)
- Phase 0の雛形(回復モデル/インターバルテーブル)を本実装に拡張
- 重量提案: ダブルプログレッション。重量ステップは器具設定の配列にスナップ
- 初回種目(ログなし)は保守的初期重量(体重比の係数テーブルを定数化)
- 全ロジックにVitestユニットテスト(境界値: ログゼロ/時間15分/全部位疲労時 を必ず含む)

### 1-4 その日のヒヤリング画面
- 時間(15/30/45/60/90)・部位(おまかせ/指定)・コンディション(3択)を3タップで入力→生成
- 生成結果画面: 種目ごとに 重量×レップ×セット+インターバル秒を表示。入れ替え/削除/追加(同部位・同器具の代替候補提示)

### 1-5 ワークアウト実行画面+インターバルタイマー
- 要件F-05の全項目。タイマー: セット完了→自動起動(設定でOFF可)、円形カウントダウン、残3秒カウント音、終了バイブ+音、±15秒、スキップ、Wake Lock
- 中断保存(アプリ再起動後に「再開しますか?」)

### 1-6 ログ保存
- 要件F-06の全項目を保存。ログタブに履歴一覧(日付・部位・所要時間・完遂率)+詳細表示
- 痛みフラグの部位は次回メニュー生成で自動回避されること(エンジン連携)

## 受け入れ条件

- [ ] iPhone実寸で「生成→実行→保存→履歴表示」が通しで動く
- [ ] タイマーが画面消灯せず動き、バイブ+音が鳴る(iOS Safariの制約で音が出ない場合はバイブ+視覚点滅で代替し、制約をhandoffに記録)
- [ ] エンジンのテストがgreen(境界値ケース含む)
- [ ] 提案重量が必ず器具設定の重量配列内の値になる
- [ ] mainへのPR作成まで(マージ判断はEiichi)
