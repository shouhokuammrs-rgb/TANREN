## 実装内容(仕様§1どおり)

### §1-1 定数(`src/constants/engine.ts`)
- `FRESHNESS_READY_THRESHOLD = 100` 新設(おまかせ選択の下限。体感調整前提のコメント付き。例: 95に緩めると端数回復を許容)
- `FRESHNESS_WARN_THRESHOLD = 50` は指定モード警告用として存続(用途コメント更新)
- `SHORTENED_NOTICE_RATIO = 0.8` 新設

### §1-2 部位選択(`selectMuscles`)
- おまかせ: `freshness >= FRESHNESS_READY_THRESHOLD` のみ選択。要求部位数未満でも**繰り上げ補充しない**
- 全部位100%未満: 従来の「最もフレッシュな1部位」フォールバックを廃止し、空選択+`isRestDay: true`
- 戻り値に `excludedRecovering: { muscle, freshness }[]` を追加(**回復が進んでいる順**にソート。仕様に順序指定がなかったための裁量)。痛みフラグ部位は理由が異なるため含めない
- 指定モードは変更なし(50%未満警告・ISS-011状態語を維持)

### §1-3 生成結果メタ(`GeneratedMenu`)
- `requestedMinutes` / `isShortened` / `excludedRecoveringMuscles` / `isRestDay` を追加
- `isShortened` = 休養日でない かつ 除外部位あり かつ `estimatedMinutes < requestedMinutes × 0.8`(境界ちょうどは短縮扱いしない・テストで固定)
- `rationale` は短縮時「今日は◯分のメニューになります。理由:△△と□□が回復中のため」、休養日「今日は休養日がおすすめです。全部位が回復中のため」を優先。文言は `MENU_COPY` に集約

### §1-4 UI(`WorkoutPage`)
- 短縮通知: メニュー上部にember-tint+molten枠のチップ+🌙(赤・⚠️なしの中立トーン)。部位列挙は3件まで「と」区切り、4件以上「A、B、Cほか」
- 休養日ビュー: メニューの代わりに「🌙 今日は休養日がおすすめです/全部位が回復中のため」カード+`FreshnessBodyMap` 再利用(全部位の%が見える)+ヒント「それでも動きたい日は「部位を選ぶ」で指定できます(軽め推奨)」+「ヒヤリングに戻る」ボタン

### §1-5 テスト(全125本green)
- **新規 `selection.test.ts`**(既存ファイルは存在しなかったため新設): 繰り上げなし/休養日/回帰/指定モード警告/痛みフラグは除外リスト対象外 の5本
- `menu.test.ts`: 旧「全部位疲労でも警告付きでメニューを返す」を「休養日提案」に更新。短縮フラグ+指定文言/回帰(充足時は通知なし)/列挙上限(3件+ほか)/`SHORTENED_NOTICE_RATIO`境界 を追加
