# デザイン探索ブリーフ: 部位別ゴールモデル(Phase 7 / DEC-013候補)

- 発行: PM / 2026-07-23
- 依頼先: Claude Design
- 成果物: HTMLモック3案(既存の探索と同様、`docs/design/` 配下想定の単一HTML)
- 必読の前提: TANRENは「鍛冶場」コンセプト(ダーク基調+溶鉄オレンジ)。既存の成長ビュー「熱の人体図」(成長=溶鉄の彩度)と世界観を揃えること

## 1. 機能の骨子(確定済み・変更不可)

- 部位ごとにゴールを3段階プリセットから選ぶ: **引き締め / しっかり / がっつり**
- 対象部位は**5つのみ**: 胸・背中・肩・腕・脚(尻・腹はv1対象外。UIに出さないか「対象外」と控えめに示すかはデザイン判断に委ねる)
- 裏の数値(目標e1RM = 体重×係数)はいつでも直接編集可能。プリセットは入口であって縛りではない
- ゴール到達時は通知→**鏡チェック**: 「満足→維持モードへ」or「物足りない→1段引き上げ」
- 成長ビュー(e1RM推移グラフ)に**ゴールラインを表示**し、あとどれくらいかが見える

## 2. 探索してほしいもの(4点セット×3案)

### 2-1. レベル別イラスト(最重要・案の個性が出るところ)
- 部位ごとのシルエット3段階(引き締め/しっかり/がっつり)。写実は不要、TANRENの世界観に合う抽象度を探ること
- 各レベルに**生活言語の説明文**を添える(例: 「Tシャツの袖から出る腕にうっすら形がある」「半袖で明らかに鍛えているとわかる」)。文言もデザイン側で提案してよい
- 注記1行を必ず添える: 「見え方は体脂肪率や骨格でも変わります」

### 2-2. ゴール設定UI
- 初回セットアップウィザード拡張 or 設定画面のどちらに置くかも含めて提案可
- 5部位×3レベルの選択が**片手・低思考**で完了すること(デザイン原則1)
- 数値直接編集への導線(上級者向け。前面に出しすぎない)
- **器具上限の注記**: 胸・背中の「がっつり」は現器具(24kgダンベル)では到達不能。選択は妨げず「現在の器具では届きません」と正直に示す表現を探ること

### 2-3. 到達チェックUI(鏡チェック)
- 通知〜判定のフロー: 「◯◯がゴールに到達しました。鏡を見て判定してください」→ 満足(維持へ)/ 物足りない(1段上げ)
- 達成の瞬間なので**祝ってよい**(デザイン原則5: PR演出に準ずる熱さ。ただし判定という選択を邪魔しない)
- 維持モードに入った部位の表現(消えない・冷たくしすぎない。「維持=完成した鉄」のようなポジティブな含意を探ること)

### 2-4. 成長ビューへのゴールライン統合
- 既存のe1RM推移グラフ(熱の人体図のフルスクリーン遷移先)にゴールラインを重ねる
- 人体図・部位カード側のゴール達成度表現(色は既存の「成長=溶鉄の彩度/回復=鈍色」の意味体系と衝突しないこと)

## 3. 制約・注意

- モックの数値は実仕様に合わせる: e1RMは片手kg表記。係数例: 胸しっかり=体重×0.50(70kgなら35kg)
- 実行画面・タイマーには触れない(スコープ外)
- 3案は方向性を変えること(例: イラスト主導/数値主導/メタファー主導、など。切り口はデザイン判断)
- 各案に1行のコンセプト名を付ける(炉心方式)
