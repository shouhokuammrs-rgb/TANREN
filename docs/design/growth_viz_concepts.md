## 3a 熱の人体図 — マップがヒーロー
人体図(FRONT/BACK)を唯一のヒーローに据え、伸び率を熱の色(暗→溶鉄→金)にエンコード。炉心の「熱=活動」の世界観を成長の意味論に直結させる。
- 部位カード: チップ列(名前+変化率)。タップで下部のグラフカードが切り替わる
- データ不足: 「冷えた鉄」=無彩色+破線。チップに「あと n 回」で次の行動を示す
- 回復予測: 最下部に固定スロットを予約。色エンコーディング(成長)との混同を避けるため分離

## 3b 数字ヒーロー — 最伸長部位が主役
炉心の「数字がヒーロー」原則を貫き、最も伸びた部位の変化率を Anton 大型数字で一点突破。人体図は補助(選択UI)に格下げ。
- 部位カード: ランキング行(順位+スパークライン+e1RM+変化率)
- データ不足: ランキングから除外し「計測中」枠に隔離。進捗ドット(1/3)で残り回数を表示 — 不確かな数字を順位に混ぜない
- 回復予測: ランキング下の別カードとして追加可能(ヒーロー領域と競合しない)

## 3c リンググリッド — 炉心の語彙を転用
実行画面のタイマーリングを成長メトリクスに転用。リング1周=「30日で+12%」の月次目標に正規化し、部位間の絶対重量差に依存しない比較を可能にする。
- 部位カード: リングカード 3×2 グリッド(中央に変化率)。タップで下部グラフ切替
- データ不足: 同じリング部品のまま破線+中央にセッション進捗(1/3)
- 回復予測: 将来的に各リングの外周へ第2リングとして重ねられる拡張性
- 注: 人体図は持たない(グリッド自体が全身一覧)。マップ必須なら 3a のマップを上部併置する拡張で対応

## 共通仕様
- e1RM はセット実績(重量×レップ)から算出。新規の記録作業なし
- 変化率 = 期間内 最初のセッション e1RM → 最新セッション e1RM
- トークン・タイポは `handoff/TANREN-handoff.md` に準拠

# 実装仕様(DEC-011 磨き込み)

## e1RM 算出式(仮置き)
**Epley式: `e1RM = 重量 × (1 + レップ ÷ 30)`**
- セッションの部位 e1RM = その部位に属する種目・全セットの e1RM の最大値
- ⚠️ 式の最終決定は PM の INS で行う(仮置き。Brzycki 等への差し替え可能なよう算出関数は1箇所に隔離すること)

## 熱の色スケール(成長マップ / 部位の伸び率)
色は常に **月換算の伸び率**(30日あたり%)でエンコードする。90日表示時は `月換算 = 実測% × 30/90`。期間を切り替えても色の意味が変わらないための正規化。

| 月換算伸び率 | 色 | 意味 |
|---|---|---|
| データ不足 | `#1C140E` +破線枠 `#3A2213`(dash 3 3) | 冷えた鉄 |
| 0% 以上 3% 未満 | `#5A2E14` | 微温 |
| 3% 以上 6% 未満 | `#8A431C` | 温 |
| 6% 以上 9% 未満 | `#C2521C` | 熱 |
| 9% 以上 12% 未満 | `#FF5C1A` (molten) +グロー | 高熱 |
| 12% 以上 | `#FFB300` (gold) +グロー | 白熱 |

- グロー: 9%以上で `drop-shadow(0 0 6px rgba(255,92,26,.6))`
- 選択中の部位: ストローク `#FFE3CC` 1.5px
- マイナス成長は現状 0% 側の色に丸める(下降の別色は導入しない — 成長ビューの意味論を「熱」に限定)

## データ不足判定
**定義: 表示期間内のセッション数が 3 回未満の部位は変化率・推移グラフを表示しない**
- 表示: マップ=冷えた鉄(上表)/ チップ=破線+「あと n 回」/ グラフ枠=「推移を表示するにはセッションがあと n 回必要です(k/3)」
- 根拠: 2点では線ではなく傾きしか描けず、double progression ではレップ増→重量増の谷で e1RM が一時的に凹むため、2点間の変化率は誤解を生みやすい。最低3点で「傾向」として提示できる
- セッションの数え方: 同一日に同部位の種目を複数実施しても1セッション

## 期間切替とフルスクリーン推移(4a)
- 期間セグメント: 30日 / 90日(ヘッダー右)。切替時: グラフ・チップの変化率は実測%、マップの色のみ月換算%
- グラフカードタップ → フルスクリーン推移へ(riseIn .35s: translateY 12px→0 + fade)。データ不足部位はタップ無効
- フルスクリーン構成: ヘッダー(部位名+✕)/ e1RM 現在値(Anton 64px)+変化率 / 拡大グラフ(補助罫線入り)/ セッション履歴(日付・e1RM・前回差。最新順・最大6件)

## ダッシュボード組み込み(4b)
- 成長カード: ミニ人体図(FRONT・30日固定)+ 上位3部位の変化率のみ。タップで成長ビューへ
- 回復予測スロット: 最下部固定(CTAの下・タブバーの上)。**色の意味を分離**する — 成長=溶鉄の彩度、回復=無彩色寄りの鈍色(バー `#8A5A3C` / 回復済み `#6B5A4C`)。molten は回復表現に使わない
- 「今日のトレを始める」CTA が常に成長カード・回復スロットより視覚優先
